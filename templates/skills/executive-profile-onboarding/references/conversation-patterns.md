# Conversation Patterns

目标：让用户用最少的字，给出最多的信息。

## 总原则

1. 先猜，再问
2. 先选项，再开放
3. 一次只推进一个决策
4. 用户回短词时，AI 负责补全结构
5. 对重要画像，至少追问到“可执行”而不是“知道个大概”

## 最省字回复法

优先使用：

- `1`
- `1 3`
- `A`
- `A 英文`
- `跳过`
- `改成 XXX`

不要默认要求：

- 长段自述
- 多题混合回答
- 一次性完整画像

## AI 先猜的标准句式

### 模式 1：猜测 + 确认

```text
我先按这个理解：
1. 你主要关心 [AI应用]
2. 更偏 [商业落地] 而不是 [底层技术]

对的话回 `1`
要改的话回 `2 + 关键词`
```

### 模式 2：给用户极短选项

```text
这块你更想要哪种？
1. 每天少量推送
2. 只在你要时再查
3. 重点方向主动推，其他按需

回数字即可
```

### 模式 3：短回答扩写

用户说：`AI`

不要收口。继续：

```text
我先缩成 3 个常见方向，你选最像的：
1. 模型/技术突破
2. 商业应用和产品机会
3. 行业格局和大公司动作

回数字即可，可多选
```

## 第 7 组知识偏好的三轮协议

### 第 1 轮：粗定位

目标：确定主题，不求精细。

建议问法：

```text
你最近最想持续跟的方向，先选大类：
1. AI/大模型
2. 商业战略
3. 组织管理
4. 技术架构
5. 行业研究

回数字即可，可多选；没有就回关键词
```

### 第 2 轮：细化维度

目标：缩到搜索可用。

建议问法：

```text
我先猜你的关注点更像下面这些，你选最像的：
1. 技术突破
2. 商业落地
3. 行业格局
4. 政策变化
5. 投资机会

再补一个深度：
A 泛看
B 持续跟踪
C 研究级

回法示例：`2 B`
```

### 第 3 轮：推送与语言

目标：形成可执行推送规则。

建议问法：

```text
最后定一下推送方式：
1. 每天主动推
2. 按需再查
3. 混合

语言：
A 中文
B 英文
C 中英都要

回法示例：`3 C`
```

## 什么情况下还要第 4 轮

只有这几种情况再追问：

- 用户选了主动推送或混合，但没给频率
- 用户的主题太大，无法转成搜索词
- 用户说“这类不用推了”，需要明确排除项

否则不要为了凑轮数而继续问。
