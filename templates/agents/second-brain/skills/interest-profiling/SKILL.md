# 技能：兴趣画像建立与维护

## 触发条件

首次使用知识管家时自动触发；或用户说"更新我的关注方向"。

## 对话目标

不是问到“有兴趣 AI”就结束，而是至少把兴趣画像问到可搜索、可推送、可排除。

默认至少 3 轮：

1. 主题粗定位
2. 维度与深度细化
3. 推送偏好定型

## 执行步骤（交互式对话）

### 第 1 轮：主题粗定位

优先给 4-6 个大类选项，让用户回数字，不要直接要求长文本。

示例：

```text
你最近最想持续跟的方向，先粗选：
1. AI/大模型
2. 商业战略
3. 组织管理
4. 技术架构
5. 行业研究

回数字即可，可多选；没有就回关键词
```

如果用户只回一个短词，例如“AI”，不要结束，继续第 2 轮。

### 第 2 轮：维度与深度细化

先做 AI 猜测，再让用户最小确认。

示例：

```text
我先猜你更想看的是：
1. 技术突破
2. 商业落地
3. 行业格局
4. 政策变化
5. 投资机会

深度再选一个：
A 泛看
B 持续跟踪
C 研究级

回法示例：`2 B`
```

这一轮还要顺手确认语言偏好，尽量让用户只补 1 个字母：

- `A 中文`
- `B 英文`
- `C 中英都要`

### 第 3 轮：推送偏好定型

不要开放式问“你希望怎么推”，直接给短选项：

```text
最后定推送方式：
1. 每天主动推
2. 按需再查
3. 混合

如果选 1 或 3，再补：
A 每天
B 每两天
C 每周

回法示例：`3 A`
```

如果还需要时间和条数，再给一轮极短补充，不要一次抛 4 个开放题。

### 第 4 轮：只在必要时补

只在以下情况继续追问：

- 主题太泛，无法形成搜索关键词
- 用户选了主动 / 混合，但没给频率或条数
- 用户提到“不想看某类”，需要补排除项

### 收口

生成 `radar/INTERESTS.md`，并把 AI 推断出的关键词、维度和深度先写成初版；不确定项标记待细化，不要因为信息不完整而卡住。

## 少字输入规则

- 能让用户回数字，就不要让用户回整句
- 用户回复很短时，AI 负责补全结构，不要抱怨信息少
- 每轮最多让用户做 1 个动作
- 先给“我先按这个理解”，再让用户确认/修改
- 允许用户用 `1`、`1 3`、`2 B`、`跳过` 这类最短格式回答

## 输出格式（radar/INTERESTS.md）

```markdown
# 知识雷达 · 兴趣画像

## 活跃主题

### [主题名称]

- 关键词：[搜索用关键词列表]
- 深度：泛泛了解 / 深度跟踪 / 研究级
- 维度：[技术/商业/政策/行业/投资]
- 语言：[中/英/双语]
- 上次互动：[日期]
- 状态：活跃 / 降温 / 暂停

## 推送偏好

- 频率：每天
- 时间：早上8:00
- 条数上限：5
- 信息源偏好：优先权威来源

## 排除项

- [用户明确说不感兴趣的方向]
```

## 维护规则

- 用户连续3次对某主题选择"阅后即焚" → 建议降温
- 用户新增兴趣点 → 即时更新
- 每月主动问一次："你的关注方向有变化吗？"
- 不要把只聊过一轮的主题当成稳定画像
